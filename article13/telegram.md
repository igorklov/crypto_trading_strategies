Фьючерсы: покупаешь КОНТРАКТ, а не актив.

| Аспект | Спот | Фьючерс |

В коде используем URL: https://testnet.binancefuture.com

Баланс: $5000 USDT виртуальных.

➖➖➖

4/10 ✅ Как проверить ключи

Сохраняете API Key и Secret Key.

Запускаете тест (Python):

```python
import hashlib, hmac, time, requests

timestamp = int(time.time() * 1000)
params = f"timestamp={timestamp}"
signature = hmac.new(SECRET.encode(), params.encode(), hashlib.sha256).hexdigest()

headers = {"X-MBX-APIKEY": API_KEY}
r = requests.get(
    f"https://testnet.binancefuture.com/fapi/v2/account?{params}&signature={signature}",
    headers=headers
)
print(r.json().get('canTrade'))
```

Ожидаемый ответ: `True` и баланс $5000 USDT.

Важно: testnet-ключи НЕ работают с live API (fapi.binance.com). И наоборот.

5/10 🏗 Архитектура: спот + фьючерсы

Добавляем в проект два новых файла:

• futures_trader.py — размещение фьючерсных ордеров
• futures_worker.py — фьючерсный воркер

Логика: те же сигналы (RSI, ADX, BB) → разные действия:

Buy → LONG (купить контракт)
Sell → SHORT (продать контракт)

Спот и фьючерсы работают ПАРАЛЛЕЛЬНО в отдельных процессах.

➖➖➖

6/10 ⚙️ Настройка плеча и размера позиции

В config.json добавляем:

```json
{
  "leverage": 3,
  "margin_fraction": 0.1,
  "symbols": {
    "BTCUSDT": { "spot": true, "futures": true },
    "ETHUSDT": { "spot": true, "futures": true }
  }
}
```

margin_fraction: 0.1 = 10% маржи на сделку.
При плече 3× это 30% от баланса в контракте — консервативно.

Правило: никогда больше 10% маржи на сделку на старте.

➖➖➖

7/10 📊 Типы сигналов для фьючерсов

На споте: buy = купить, sell = продать (что есть).

На фьючерсах:

• Buy + ADX > 25 + RSI < 20 → LONG
• Sell + ADX > 25 + RSI > 80 → SHORT
• RSI вернулся к 50 + ADX упал → CLOSE
• Trailing stop -3% от максимума → CLOSE
• Take-profit +2.5% → CLOSE

Фундаментальный анализ (FA) блокирует LONG при негативном сентименте — как на споте.

8/10 🛑 Безопасность (абсолютные правила)

1. Неделя тестов на testnet перед live
2. Плечо ≤ 3× на старте
