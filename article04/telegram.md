Что если вы придумали торговую стратегию, но не хотите тестировать её на реальных деньгах? Ответ — **бектестинг**.

Бектестинг позволяет промоделировать стратегию на исторических данных и оценить её эффективность *до* того, как вы рискнете капиталом.

В этой статье мы построим полный конвейер бектестинга для крипторынка, а затем подключим к нему LLM‑агента, чтобы он анализировал результаты и предлагал улучшения.

---

🔧 Шаг 1. Подготовка исторических данных

Для бектеста нужны исторические свечи. Используем API Binance, чтобы скачать данные за нужный период.

```python
from binance.client import Client
import pandas as pd

client = Client(api_key, api_secret)
klines = client.get_historical_klines(
    symbol="BTCUSDT",
    interval=Client.KLINE_INTERVAL_1HOUR,
    start_str="2024-01-01",
    end_str="2024-12-31"
)
df = pd.DataFrame(klines, columns=['open_time', 'open', 'high', 'low', 'close', 'volume', 'close_time', 'quote_asset_volume', 'number_of_trades', 'taker_buy_base_asset_volume', 'taker_buy_quote_asset_volume', 'ignore'])
df['close'] = df['close'].astype(float)
```

Полученный DataFrame сохраняем в CSV — он понадобится для многократных тестов.

---

📈 Шаг 2. Простая стратегия для примера

Возьмем классическую стратегию на пересечении скользящих средних:

- **Покупаем**, когда быстрая SMA (20 периодов) пересекает медленную SMA (50) снизу вверх.
- **Продаем**, когда быстрая SMA пересекает медленную сверху вниз.

```python
df['sma20'] = df['close'].rolling(20).mean()
df['sma50'] = df['close'].rolling(50).mean()
df['signal'] = 0
df.loc[df['sma20'] > df['sma50'], 'signal'] = 1
df.loc[df['sma20'] < df['sma50'], 'signal'] = -1
```

Сигнал `1` — покупка, `-1` — продажа, `0` — нейтрально.

---

🤖 Шаг 3. Логика бектестинга

Теперь симулируем сделки: начинаем с определённого капитала, следуем сигналам, учитываем комиссию Binance (0.1%).

```python
capital = 10000.0  # стартовый капитал в USDT
position = 0.0
trades = []

for i in range(1, len(df)):
    if df['signal'].iloc[i] == 1 and position == 0:
        # покупаем на весь капитал
        price = df['close'].iloc[i]
        position = capital / price
        capital = 0.0
        trades.append(('buy', price))
    elif df['signal'].iloc[i] == -1 and position > 0:
        # продаём всю позицию
        price = df['close'].iloc[i]
        capital = position * price * (1 - 0.001)  # комиссия
        position = 0.0
        trades.append(('sell', price))
```

В конце рассчитываем итоговую доходность, максимальную просадку, Sharpe ratio.

2. **Кривая капитала** — как менялся бы наш баланс во времени.

🧠 Шаг 5. Интеграция с OpenClaw — создание skill

Теперь упакуем бектестинг в skill OpenClaw, чтобы LLM‑агент мог запускать его по запросу.

Структура skill:
- `backtest.py` — основной скрипт с логикой тестирования
- `SKILL.md` — описание skill, параметры, примеры запросов
- `requirements.txt` — зависимости

**Пример запроса к агенту:**

> «Протестируй стратегию SMA‑20/50 на BTC за 2024 год и покажи отчёт.»

Агент запускает skill, получает результаты и отправляет их в Telegram.

---

⚙️ Шаг 6. Настройка регулярного запуска

Чтобы не запускать бектест вручную, настраиваем автоматический запуск:

- **Вариант 2:** cron OpenClaw — используем встроенный планировщик (см. статью №2).
