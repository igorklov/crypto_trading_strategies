🚀 День 9: Масштабирование агентов — управление множеством пар, базы данных и панели мониторинга

Один агент работает с парой BTC/USDT? Отлично. А если нужно мониторить 20 пар, хранить историю, визуализировать метрики и автоматически перезапускаться после сбоев? Пора строить систему.

👇 Далее: архитектура масштабированной системы.

---

2/12
🏗️ Архитектура масштабированной системы

• **Агенты‑воркеры** — отдельные процессы, каждый мониторит 1‑2 пары, вычисляет индикаторы, отправляет сигналы.
• **Диспетчер** — центральный процесс, принимает решения о размещении ордеров (риск‑менеджмент, позиция).
• **База данных** — SQLite/PostgreSQL, хранит свечи, индикаторы, сигналы, ордера, снимки портфеля.
• **Панель мониторинга** — Grafana + Prometheus или простой Flask‑интерфейс.
• **Оркестратор** — systemd службы + watchdog, перезапускает упавшие компоненты.
• **Логирование и алертинг** — единый лог + Telegram‑уведомления.

Цель: отказоустойчивая, наблюдаемая инфраструктура 24/7.

3/12
👷 Мультипроцессный мониторинг: запуск десятков воркеров

GIL в Python ограничивает потоки для CPU‑bound задач. Для полной изоляции используем **процессы** (`multiprocessing`). Каждый воркер работает независимо, падение одного не влияет на остальных.

Пример класса Worker (упрощённо):

```python
class Worker(Process):
    def __init__(self, symbol, interval, config, signal_queue):
        super().__init__()
        self.symbol = symbol
        self.interval = interval
        self.signal_queue = signal_queue
```

Воркер получает свечи, вычисляет индикаторы (RSI, SMA), генерирует сигналы и отправляет их в очередь диспетчеру.

4/12
⚙️ Менеджер процессов

Менеджер запускает, останавливает и перезапускает воркеров на основе конфигурации.

```python
class ProcessManager:
    def start_all(self):
        for symbol, params in config['symbols'].items():
            w = Worker(symbol, params['interval'], config, queue)
            w.start()
            self.workers[symbol] = w
```

Конфигурация в YAML/JSON:

```yaml
symbols:
  BTCUSDT:
    interval: 15m
    strategy: rsi_sma
  ETHUSDT:
    interval: 1h
    strategy: macd
```

5/12
🗄️ База данных SQLite: храним историю и метрики

SQLite — лёгкая, встроенная БД, идеальна для начала.

Создаём таблицы:
• `candles` — свечи (open, high, low, close, volume)
• `indicators` — значения индикаторов (RSI, SMA, MACD, Bollinger Bands)
• `signals` — сигналы (buy/sell, сила, время)
• `orders` — ордера (ID, сторона, тип, статус)
• `portfolio_snapshots` — снимки портфеля (баланс, эквити)

Функция `save_candle()` сохраняет каждую свечу с уникальным ключом `(symbol, interval, open_time)`.

📊 Панель мониторинга: два варианта

**Вариант A (продвинутый): Prometheus + Grafana**

7/12
🐕‍🦺 Оркестрация: systemd службы и watchdog

Каждый компонент — отдельная systemd служба:

```ini
[Unit]
Description=Trading Process Manager
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/python3 /path/to/process_manager.py
Restart=always
RestartSec=10
```

**Watchdog** — скрипт, который каждую минуту проверяет `systemctl is‑active` и перезапускает упавшие службы, отправляя уведомление в Telegram.
