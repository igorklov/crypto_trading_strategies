## Введение

За 13 статей мы построили торговую систему, которая анализирует технические индикаторы (RSI, SMA, MACD, Bollinger Bands, ADX), фундаментальные факторы через LLM и управляет позициями на споте и фьючерсах. В 14-й статье разобрали стратегию на уровнях Фибоначчи. Но есть один классический инструмент технического анализа, который мы пока не автоматизировали — **трендовые линии**.

Трендовые линии — это не просто «линии на графике». Это визуализация рыночной структуры:

- **Линия восходящего тренда** (uptrend line) — соединяет последовательные более высокие минимумы (higher lows). Пока цена выше этой линии, тренд в силе.
- **Линия нисходящего тренда** (downtrend line) — соединяет последовательные более низкие максимумы (lower highs). Пока цена ниже — тренд в силе.
- **Пробой линии (breakout)** — один из самых сильных торговых сигналов. Когда цена пробивает трендовую линию — тренд, скорее всего, сменился.

Сегодня мы добавим в нашу систему **автоматическое построение трендовых линий и детекцию breakout'ов**, интегрируем это с LLM-агентами для контекстного анализа пробоев и научимся отличать ложные пробои от истинных.

## 1. Почему трендовые линии важны для автоматической торговли?

Большинство индикаторов (RSI, MACD) — опережающие (leading) или запаздывающие (lagging). Они следуют за ценой и подтверждают тренд постфактум. Трендовые линии дают другой тип информации:

| Инструмент | Тип сигнала | Задержка |
|:-----------|:------------|:--------:|
| RSI | Опережающий (перекупленность/перепроданность) | Свеча |
| MACD | Запаздывающий (кросс линий) | Несколько свечей |
| SMA | Запаздывающий (кросс цены и средней) | Период SMA |
| **Трендовая линия** | **Сигнал слома структуры** | **1 свеча** |
| **Пробой трендовой** | **Смена тренда** | **Моментально** |

**Преимущество трендовых линий в автоматической системе:**

1. **Раннее обнаружение смены тренда** — пробой линии даёт сигнал раньше, чем SMA или MACD подтвердят разворот.
2. **Фильтр сигналов** — если цена пробила восходящий тренд, любой buy-сигнал от RSI/ADX имеет меньший вес.
3. **Уровни для take-profit и stop-loss** — трендовые линии можно использовать как динамические уровни: TP на следующей линии сопротивления, SL под линией поддержки.
4. **Контекст для LLM** — LLM может анализировать не только RSI=30, но и «цена пробила 3-месячную трендовую линию» — это гораздо более содержательное описание ситуации.

## 2. Алгоритм автоматического построения трендовых линий

### 2.1. Поиск ключевых точек (swing highs / swing lows)

Основа трендовой линии — экстремумы. Нам нужно найти значимые максимумы и минимумы на ценовом графике.

Простой алгоритм поиска swing-точек:

```python
def find_swing_points(highs, lows, window=5):
    """
    Находит swing high и swing low точки.
    window — сколько свечей слева и справа должно быть ниже/выше.
    """
    length = len(highs)
    swing_highs = np.full(length, np.nan)
    swing_lows = np.full(length, np.nan)
    
    for i in range(window, length - window):
        if highs[i] == max(highs[i-window:i+window+1]):
            swing_highs[i] = highs[i]
        if lows[i] == min(lows[i-window:i+window+1]):
            swing_lows[i] = lows[i]
    
    return swing_highs, swing_lows
```

Чем больше `window`, тем меньше точек мы находим, но они более значимы. Для 15-минутного графика `window=5` — хороший старт (25 свечей = ~6 часов). Для 4-часового — `window=3` достаточно (12 часов).

### 2.2. Построение линий и проверка касаний (bounce test)

Трендовая линия строится по **двум точкам**, а **третья точка** подтверждает её значимость. Алгоритм перебирает все пары swing-точек, для каждой вычисляет наклон и intercept, затем считает, сколько точек ложится в пределах допуска (обычно 3%) от линии. Линии с 3+ касаниями считаются валидными, с 4+ — сильными.

```
Линия с 2 касаниями = шум
Линия с 3 касаниями = валидный уровень
Линия с 4+ касаниями = сильный уровень (торгуемый)
```

### 2.3. Обнаружение пробоя (breakout detection)

Самый важный сигнал — когда цена пересекает трендовую линию. Различаем два типа:

- **🟢 Bullish breakout** — цена была ниже downtrend линии, стала выше. Медвежий тренд сломлен.
- **🔴 Bearish breakout** — цена была выше uptrend линии, стала ниже. Бычий тренд сломлен.

Базовая логика проверки:

```python
def detect_breakout(closes, timestamps, trend_lines):
    """Детектит пробой на последней свече."""
    current_price = closes[-1]
    prev_price = closes[-2]
    current_time = timestamps[-1]
    breakouts = []
    
    for line in trend_lines:
        expected = line['slope'] * current_time + line['intercept']
        prev_expected = line['slope'] * timestamps[-2] + line['intercept']
        
        if line['is_uptrend'] and prev_price > prev_expected and current_price < expected:
            breakouts.append({'type': 'bearish_breakout', ...})
        elif not line['is_uptrend'] and prev_price < prev_expected and current_price > expected:
            breakouts.append({'type': 'bullish_breakout', ...})
    
    return breakouts
```

### 2.4. Фильтрация ложных пробоев

До 60% пробоев — ложные. Цена может «проткнуть» линию и вернуться обратно. Используем четыре критерия:

1. **Моментум > 0.5× ATR** — цена должна реально пробить, а не просто коснуться
2. **Объём > 1.5× среднего** — пробив на объёме гораздо надёжнее
3. **Сила линии > 60%** — линия должна иметь 3+ касания
4. **Norm momentum > 1.0× ATR** — усиленный критерий моментума

Сигнал считается валидным при выполнении минимум 2 из 4 критериев.

## 3. Интеграция с существующей системой

### Архитектура

```
Данные → TA (RSI, ADX, BB) → TrendLineAnalyzer → Комбинатор → Ордер
                                  ↓
                            LLM оценка пробоя
```

Новый модуль `TrendLineAnalyzer` работает как прослойка между индикаторами и логикой принятия решений. Он:

1. Принимает DataFrame с OHLCV-данными
2. Находит swing-точки с заданным window
3. Строит валидные uptrend и downtrend линии
4. Детектит пробои на последней свече
5. Фильтрует ложные пробои через ATR и объём
6. Определяет текущий тренд (uptrend / downtrend / ranging)

Подключение в существующий worker — буквально 5 строк:

```python
from trend_line_analyzer import TrendLineAnalyzer

trend_analyzer = TrendLineAnalyzer(symbol, interval, config.get('trend_lines', {}))
result = trend_analyzer.analyze(df)
signals['breakout'] = result.get('breakouts', [])
signals['trend'] = result.get('current_trend', 'undefined')
```

### Комбинирование сигналов: TA + TL + FA

Финальное решение учитывает три слоя с разными весами:

| Слой | Вес | Что анализирует |
|:-----|:---:|:----------------|
| Технические индикаторы (TA) | 40% | RSI, ADX, Bollinger Bands |
| Трендовые линии (TL) | 35% | Пробои, текущий тренд |
| Фундаментальный анализ (FA/LLM) | 25% | Новости, сентимент рынка |

**Примеры комбинированных сигналов:**

| TA | TL | FA | Решение | Пояснение |
|:---|:---|:---|:--------|:----------|
| RSI=25 (buy) | Пробой downtrend ↑ | Позитивный | **BUY** ✅ | Все три слоя совпадают |
| RSI=75 (sell) | Пробой uptrend ↓ | Негативный | **SELL** ✅ | То же |
| RSI=25 (buy) | Uptrend intact | Негативный | **HOLD** ⏸ | FA блокирует buy |
| RSI=70 (sell) | Пробой downtrend ↑ | Позитивный | **BUY** ✅ | Breakout перевешивает перекупленность |

## 4. LLM-агент для контекстного анализа пробоев

Не все пробои одинаково полезны. LLM может оценить контекст, который числовой алгоритм не видит:

```
Промпт для LLM:
- Тип пробоя (bullish/bearish)
- Моментум и уверенность
- Сила трендовой линии (количество касаний)
- RSI, ADX, объём
- 24h изменение цены

Ответ LLM: {"score": от -10 до +10, "explanation": "..."}
```

**Пример 1:** BTC пробивает uptrend на 4H графике

```
LLM: {"score": 7, "explanation": "Пробой подтверждён объёмом (+80% от среднего),
      линия имела 4 касания. RSI снижается, ADX > 25.
      Высокая вероятность смены тренда."}
```

**Пример 2:** ETH касается downtrend, но объём низкий

```
LLM: {"score": -3, "explanation": "Пробой на низком объёме (0.6× от среднего),
      линия построена всего по 2 точкам. Скорее всего, ложный сигнал —
      цена вернётся под линию."}
```

LLM здесь выступает как фильтр второго уровня: после числовой фильтрации (ATR + объём) LLM даёт контекстную оценку. В бектесте это повысило win rate с 52% до 71%.

## 5. Результаты бектестинга

Проверили стратегию «покупать при bullish breakout, продавать при bearish breakout» на BTCUSDT, 1H, за 30 дней:

| Конфигурация фильтров | Сделок | Win Rate | Возврат |
|:----------------------|:------:|:--------:|:-------:|
| Без фильтров | 47 | 34% | −12.4% |
| Объём + ATR | 18 | 52% | +4.7% |
| Все фильтры + LLM | 8 | 71% | +11.2% |

Фильтры резко снижают количество сделок (с 47 до 8), но качество сигналов вырастает в 2 раза. Для консервативной торговли это идеальный профиль.

**Параметры для оптимизации:**

| Параметр | Диапазон | По умолчанию | Влияние |
|:---------|:---------|:------------:|:--------|
| `swing_window` | 3–10 | 5 | Больше → меньше линий, но надёжнее |
| `min_touches` | 2–5 | 3 | Меньше → больше сигналов, но шумнее |
| `max_gap_pct` | 1%–5% | 3% | Допуск при проверке касаний |
| `breakout_threshold_atr` | 0.3–1.0 | 0.5 | Во сколько ATR цена должна отойти |
| `volume_multiplier` | 1.0–3.0 | 1.5 | Во сколько раз объём выше среднего |

## 7. Преимущества и ограничения

### Преимущества

| Аспект | Описание |
|:-------|:---------|
| **Ранние сигналы** | Пробой трендовой линии даёт сигнал раньше, чем SMA или MACD |
| **Фильтр шума** | Трендовые линии отсекают ложные сигналы индикаторов в боковике |
| **Контекст для LLM** | LLM получает структуру рынка, а не просто числа |
| **Динамические уровни** | Линии можно использовать как trailing SL/TP |
| **Мультитаймфрейм** | Анализ на 1H + 4H + 1D даёт полную картину |

### Ограничения

| Аспект | Описание |
|:-------|:---------|
| **Субъективность** | Алгоритм никогда не найдёт те же линии, что трейдер-человек |
| **Ложные пробои** | До 60% пробоев могут быть ложными без фильтрации по объёму и ATR |
| **Зависимость от window** | Размер окна swing-точек сильно влияет на результат |
| **Плохо в боковике** | В диапазоне 1–2% линии строятся плохо |
| **Историческая привязка** | Линии статичны, пока не появится новая точка |
