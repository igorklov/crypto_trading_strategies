🎯 1/11 Зачем риск‑менеджмент LLM‑агенту?

Человек‑трейдер может «закрыть глаза» на просадку или войти в сделку на эмоциях. Агент — нет. Но если вы не настроите правила управления рисками, агент может слить депозит так же эффективно, как и человек.

В этом выпуске покажу три уровня защиты:
1. Позиционный риск‑менеджмент (на одной сделке)
2. Портфельный риск‑менеджмент (на всём капитале)
3. Мониторинг просадки (drawdown) в реальном времени

---

📊 2/11 Позиционный риск‑менеджмент

Это правила, которые применяются к каждой отдельной сделке. Основные параметры:

• **Размер позиции** — какой % капитала использовать в одной сделке
• **Стоп‑лосс** — автоматическое закрытие при заданном убытке
• **Тейк‑профит** — фиксация прибыли при достижении цели
• **Риск/прибыль** — соотношение R:R (например, 1:3)

Пример кода на Python:

```python
def position_risk_management(capital, risk_per_trade=0.02, stop_loss_pct=0.05):
    position_size = capital * risk_per_trade / stop_loss_pct
    return position_size
```

📈 3/11 Портфельный риск‑менеджмент

Контролирует совокупный риск по всем открытым позициям. Даже если каждая сделка рискованна на 2%, одновременно открытые 10 сделок могут дать риск 20% капитала.

Правила портфельного уровня:
• **Максимум открытых позиций** (например, 5)
• **Максимальный совокупный риск** (например, 10% капитала)
• **Корреляция активов** — не открывать позиции в сильно коррелирующих парах
• **Хеджирование** — открытие противоположных позиций для снижения риска

📉 4/11 Мониторинг просадки (drawdown)

Просадка — это снижение капитала от предыдущего максимума. Если агент допустил просадку 20%, это сигнал пересмотреть стратегию.

Код для отслеживания просадки в реальном времени:

```python
import pandas as pd

def calculate_drawdown(equity_series):
    rolling_max = equity_series.expanding().max()
    drawdown = (equity_series - rolling_max) / rolling_max * 100
    return drawdown.min()  # максимальная просадка в %
```

Интегрируйте этот мониторинг в вашего агента, чтобы получать алерты при превышении порога (например, –15%).

🛡️ 5/11 Интеграция с OpenClaw

OpenClaw позволяет вынести правила риск‑менеджмента в отдельный навык (skill). Агент будет обращаться к нему перед каждой сделкой.

Пример конфигурации в OpenClaw:

```yaml
risk_rules:
  max_position_size: 0.1      # 10% капитала на сделку
  max_portfolio_risk: 0.15    # 15% совокупного риска
  max_drawdown: -20           # стоп при –20% просадки
  correlation_threshold: 0.7   # не торговать коррелирующими активами
```

⚙️ 6/11 Практический модуль: PositionRiskManager

Приведу готовый класс, который вы можете встроить в своего агента:

```python
class PositionRiskManager:
    def __init__(self, capital, risk_per_trade=0.02, max_positions=5):
        self.capital = capital
        self.risk_per_trade = risk_per_trade
        self.max_positions = max_positions
        self.open_positions = []
    
    def can_open_position(self, symbol, stop_loss_pct):
        if len(self.open_positions) >= self.max_positions:
            return False, "Достигнут лимит открытых позиций"
        
        position_size = self.capital * self.risk_per_trade / stop_loss_pct
        if position_size > self.capital * 0.1:  # не более 10% капитала
            return False, "Превышен максимальный размер позиции"
        
        return True, position_size
```

📝 7/11 Практический модуль: PortfolioRiskManager

Класс для контроля совокупного риска:

```python
class PortfolioRiskManager:
    def __init__(self, capital, max_total_risk=0.10):
        self.capital = capital
        self.max_total_risk = max_total_risk
        self.positions = {}
    
    def add_position(self, symbol, position_size, stop_loss_pct):
        risk_amount = position_size * stop_loss_pct
        total_risk = sum(p['risk_amount'] for p in self.positions.values())
        
        if total_risk + risk_amount > self.capital * self.max_total_risk:
            return False, "Превышен совокупный риск портфеля"
        
        self.positions[symbol] = {
            'size': position_size,
            'risk_amount': risk_amount
        }
        return True, "Позиция добавлена"
```

🔍 9/11 Как тестировать правила риск‑менеджмента?

Используйте исторические данные и симуляцию:

1. Загрузите данные за прошлый год
2. Прогоните свою стратегию через бэктест
3. Примените правила риск‑менеджмента
4. Сравните результаты:
   • Без риск‑менеджмента → высокая доходность, но огромные просадки
   • С риск‑менеджментом → меньшая доходность, но стабильный рост капитала

Именно второй вариант позволяет «выжить» на рынке в долгосрочной перспективе.
