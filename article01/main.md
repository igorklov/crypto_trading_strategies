# Установка и настройка OpenClaw: первый шаг к автономному трейдингу

## Введение

Вы уже умеете писать скрипты на Python, знаете основы работы с биржами, но хотите перейти от ручного трейдинга к автономным системам, которые работают 24/7. Первый шаг — выбрать платформу для создания и управления LLM‑агентами.

В этой серии статей мы будем использовать **OpenClaw** — open‑source платформу, которая позволяет запускать агентов с доступом к инструментам (API бирж, веб‑поиск, выполнение кода) и организовывать их взаимодействие. OpenClaw работает на вашем сервере, не требует облачных подписок и даёт полный контроль над логикой и данными.

В этой статье мы развернём OpenClaw с нуля на Linux‑сервере (Debian/Ubuntu), настроим окружение и убедимся, что система готова к созданию первого трейдинг‑агента.

## Что вам понадобится

- Сервер с **Linux** (Debian 12+, Ubuntu 22.04+) или собственная машина (можно даже Raspberry Pi).
- Установленный **Node.js 22+** (рекомендуется 22.16.0 или выше) и **npm**.
- Базовое знакомство с командной строкой.
- 15–20 минут времени.

## Шаг 1. Установка Node.js и npm

Если Node.js ещё не установлен, выполните следующие команды (для Debian/Ubuntu):

```bash
# Обновляем список пакетов
sudo apt update

# Устанавливаем curl (если нет)
sudo apt install -y curl

# Добавляем официальный репозиторий Node.js (версия 22.x)
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -

# Устанавливаем Node.js и npm
sudo apt install -y nodejs
```

Проверяем установку:

```bash
node --version   # Должно быть v22.x.x (рекомендуется ≥22.16.0)
npm --version    # Должно быть 10.x.x или 11.x.x
```

**Важно:** OpenClaw требует Node.js версии 22.16.0 или выше. Если у вас уже установлена более старая версия (например, 20.x), обновите её, выполнив команды выше.

---

## Шаг 2. Установка OpenClaw

OpenClaw распространяется как npm‑пакет. Установим его глобально:

```bash
sudo npm install -g openclaw
```

Эта команда скачает пакет и поместит исполняемый файл `openclaw` в `/usr/local/bin/`.

Проверим, что установка прошла успешно:

```bash
openclaw --version
```

Вы должны увидеть номер версии (например, `2026.3.22`).

**Если команда `openclaw` не найдена**, выполните одно из следующих действий:

1. **Сбросьте кэш команд bash:**
   ```bash
   hash -r
   ```
   Затем повторите `openclaw --version`.

2. **Если это не помогло**, проверьте, создался ли симлинк:
   ```bash
   ls -la /usr/local/bin/openclaw
   ```
   Если симлинк отсутствует, создайте его вручную:
   ```bash
   sudo ln -sf /usr/lib/node_modules/openclaw/openclaw.mjs /usr/local/bin/openclaw
   sudo chmod +x /usr/local/bin/openclaw
   ```

3. **Если после установки вы видите версию `0.0.1`** — это «пустой» placeholder‑пакет, который устанавливается при несовместимой версии Node.js. Удалите его и установите заново после обновления Node.js до версии ≥22.16.0:
   ```bash
   sudo npm uninstall -g openclaw
   sudo npm install -g openclaw
   ```

## Шаг 3. Первоначальная настройка (onboard)

Вместо устаревшей команды `init` используйте `onboard` — она проведёт вас через интерактивную настройку OpenClaw под ваши нужды.

```bash
# Запускаем мастер настройки
openclaw onboard
```

В процессе вам будут заданы следующие вопросы. Выбирайте варианты, как показано ниже, чтобы настроить DeepSeek Reasoner в качестве модели по умолчанию:

1. **`I understand this is personal‑by‑default and shared/multi‑user use requires lock‑down. Continue?`**  
   → Выберите **`Yes`**.

2. **`Setup mode`**  
   → Выберите **`Manual`** (ручная настройка даёт больше контроля).

3. **`What do you want to set up?`**  
   → Выберите **`Local gateway (this machine)`**.

4. **`Workspace directory`**  
   → Оставьте по умолчанию: **`/root/.openclaw/workspace`**.

5. **`Model/auth provider`**  
   → Выберите **`Custom Provider`**.

6. **`API Base URL`**  
   → Введите: **`https://api.deepseek.com`**.

7. **`How do you want to provide this API key?`**  
   → Выберите **`Paste API key now`**.

8. **`API Key (leave blank if not required)`**  
   → Вставьте ваш API‑ключ DeepSeek (получить можно на [platform.deepseek.com](https://platform.deepseek.com/)).

9. **`Endpoint compatibility`**  
   → Выберите **`OpenAI‑compatible`**.

10. **`Model ID`**  
    → Введите: **`deepseek‑reasoner`**.

11. **`Verification successful.`**  
    → Система проверит подключение. Если ключ корректный, вы увидите это сообщение.

12. **`Endpoint ID`**  
    → Система предложит что‑то вроде **`custom‑api‑deepseek‑com`**. Оставьте как есть.

13. **`Model alias (optional)`**  
    → Введите: **`deepseek`** (или оставьте пустым).

14. **`Gateway port`**  
    → Оставьте по умолчанию: **`18789`**.

После завершения настройки OpenClaw создаст все необходимые файлы конфигурации в `~/.openclaw/config.json` и подготовит workspace.

## Шаг 4. Запуск Gateway

Gateway должен работать как фоновая служба. Запустим его:

```bash
openclaw gateway start
```

Проверим статус:

```bash
openclaw gateway status
```

Если всё хорошо, вы увидите `Gateway is running`. Обратите внимание: по умолчанию Gateway использует порт **18789** (вы могли изменить его в процессе onboard). Чтобы убедиться, что сервис слушает правильный порт, выполните:

```bash
ss -tlnp | grep 18789
```

Также можно проверить лог Gateway:

```bash
openclaw gateway log --tail
```

---

## Шаг 5. Проверка работоспособности

Убедимся, что OpenClaw отвечает. Отправим простую команду:

```bash
openclaw status
```

Вывод должен содержать информацию о версии, статусе Gateway, количестве сессий.

Также можно запустить тестовый агент в интерактивном режиме:

```bash
openclaw run "Напиши 'Hello, World!' на Python и выполни код"
```

Если агент успешно выполнит запрос — система работает корректно.

## Шаг 6. Настройка переменных окружения (опционально)

Для работы с внешними API (например, Binance) позже потребуется хранить ключи в безопасном месте. OpenClaw поддерживает файл `.env` в workspace.

Создадим его:

```bash
cd ~/.openclaw/workspace
nano .env
```

Добавим пример переменной (пока placeholder):

```env
# Ключи API бирж (будем добавлять позже)
BINANCE_API_KEY=your_api_key_here
BINANCE_API_SECRET=your_api_secret_here

# Ключ для Telegram‑бота (для уведомлений)
TELEGRAM_BOT_TOKEN=your_bot_token
```

Сохраните файл (`Ctrl+O`, `Enter`, `Ctrl+X`). OpenClaw автоматически загрузит эти переменные при запуске агентов.

## Шаг 7. Знакомство с архитектурой OpenClaw

Прежде чем двигаться дальше, полезно понять основные компоненты:

1. **Gateway** — центральный сервис, управляет агентами, предоставляет API.
2. **Agents** — экземпляры LLM с доступом к инструментам (чтение файлов, выполнение кода, вызов API).
3. **Workspace** — директория, где хранятся все файлы проекта (скрипты, конфиги, логи).
4. **Skills** — готовые модули для типовых задач (например, `healthcheck`, `video‑frames`). Мы будем создавать свои skills для трейдинга.
5. **Tools** — инструменты, которые агент может использовать (браузер, запросы к API, шелл).

## Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| `openclaw: command not found` | Убедитесь, что npm установил пакет глобально (`npm list -g openclaw`). Добавьте `/usr/local/bin` в `PATH`. Если команда всё ещё не найдена, выполните `hash -r` или создайте симлинк вручную (см. Шаг 2). |
| Gateway не запускается (`port already in use`) | Измените порт в `~/.openclaw/config.json` (`gateway.port`) или освободите порт 18789 (по умолчанию). |
| Ошибка «Permission denied» при записи в workspace | Проверьте права на директорию `~/.openclaw`. Выполните `sudo chown -R $USER:$USER ~/.openclaw`. |
| Агент не может выполнить код | Проверьте, установлен ли Python3 (`python3 --version`). Установите: `sudo apt install python3`. |
| Установился placeholder‑пакет `openclaw@0.0.1` | Это происходит при несовместимой версии Node.js (<22.16.0). Обновите Node.js до версии ≥22.16.0, удалите пакет (`npm uninstall -g openclaw`) и установите заново. |
| Команда `openclaw` не работает после установки | Выполните `hash -r` или перезапустите терминал. Если симлинк отсутствует, создайте его: `sudo ln -sf /usr/lib/node_modules/openclaw/openclaw.mjs /usr/local/bin/openclaw` и `sudo chmod +x /usr/local/bin/openclaw`. |
