🚀 *Статья 1: «Установка и настройка OpenClaw: первый шаг к автономному трейдингу»*

В первой статье мы установим OpenClaw — open‑source платформу для создания и управления LLM‑агентами.

OpenClaw работает на вашем сервере, не требует облачных подписок и даёт полный контроль над логикой и данными.

---

🔧 *Что понадобится:*

- Сервер с Linux (Debian/Ubuntu) или своя машина.
- Установленные Node.js 22+ (рекомендуется ≥22.16.0) и npm.
- 15–20 минут времени.

---

📦 *Шаг 1: Установка Node.js и npm*

Команды для Debian/Ubuntu:

```bash
sudo apt update
sudo apt install -y curl
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
```

Проверяем:

```bash
node --version
npm --version
```

---

📦 *Шаг 2: Установка OpenClaw*

Устанавливаем глобально через npm:

```bash
sudo npm install -g openclaw
```

Проверяем версию:

```bash
openclaw --version
```

---

📂 *Шаг 3: Первоначальная настройка (onboard)*

Вместо `init` используйте `onboard`:

```bash
openclaw onboard
```

Ответьте на вопросы (выбирайте варианты для DeepSeek):
- **Continue?** → `Yes`
- **Setup mode** → `Manual`
- **What to set up?** → `Local gateway`
- **Workspace directory** → по умолчанию
- **Model/auth provider** → `Custom Provider`
- **API Base URL** → `https://api.deepseek.com`
- **API key** → вставьте ключ DeepSeek (platform.deepseek.com)
- **Endpoint compatibility** → `OpenAI‑compatible`
- **Model ID** → `deepseek‑reasoner`
- **Gateway port** → `18789`

---

⚙️ *Шаг 4: Запуск Gateway*

Gateway — фоновый сервис, который управляет агентами.

```bash
openclaw gateway start
openclaw gateway status
```

Если всё хорошо, увидите `Gateway is running`. По умолчанию Gateway использует порт **18789**.

---

🧪 *Шаг 5: Проверка работоспособности*

Проверяем, что OpenClaw отвечает:

```bash
openclaw status
```

Запускаем тестового агента:

```bash
openclaw run "Напиши 'Hello, World!' на Python и выполни код"
```

Если агент выполнил запрос — система работает.

---

🔐 *Шаг 6: Настройка переменных окружения (опционально)*

Создаём файл `.env` для хранения API‑ключей:

```bash
cd ~/.openclaw/workspace
nano .env
```

Добавляем placeholder‑переменные для Binance и Telegram.

---

🏗️ *Шаг 7: Знакомство с архитектурой OpenClaw*

Кратко о компонентах:

- Gateway — центральный сервис.
- Agents — экземпляры LLM с доступом к инструментам.
- Workspace — директория проекта.
- Skills — готовые модули для типовых задач.
- Tools — инструменты (браузер, API, шелл).

🛠️ *Возможные проблемы и решения*

- `openclaw: command not found` → убедитесь, что пакет установлен глобально; выполните `hash -r` или создайте симлинк вручную.
- Gateway не запускается (`port already in use`) → измените порт в конфиге или освободите порт 18789 (по умолчанию).
- Ошибка «Permission denied» → проверьте права на `~/.openclaw`.
- Агент не может выполнить код → установите Python3.
- Установился placeholder‑пакет `openclaw@0.0.1` → обновите Node.js до ≥22.16.0, удалите и установите заново.
- Команда `openclaw` не работает после установки → выполните `hash -r` или создайте симлинк.
