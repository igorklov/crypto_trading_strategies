🚀 *СТАТЬЯ 1/12*  
*LLM‑агенты для трейдинга: практическое руководство на Python + OpenClaw*

📌 *Установка и настройка OpenClaw: первый шаг к автономному трейдингу*

Вы уже умеете писать скрипты на Python, знаете основы работы с биржами, но хотите перейти от ручного трейдинга к автономным системам, которые работают 24/7. Первый шаг — выбрать платформу для создания и управления LLM‑агентами.

В этой серии статей мы будем использовать **OpenClaw** — open‑source платформу, которая позволяет запускать агентов с доступом к инструментам (API бирж, веб‑поиск, выполнение кода) и организовывать их взаимодействие. OpenClaw работает на вашем сервере, не требует облачных подписок и даёт полный контроль над логикой и данными.

---

🔧 *Что вам понадобится*

- Сервер с **Linux** (Debian 12+, Ubuntu 22.04+) или собственная машина (можно даже Raspberry Pi).
- Установленный **Node.js 22+** (рекомендуется ≥22.16.0) и **npm**.
- Базовое знакомство с командной строкой.
- 15–20 минут времени.

---

📦 *Шаг 1. Установка Node.js и npm*

Если Node.js ещё не установлен, выполните команды (для Debian/Ubuntu):

```bash
sudo apt update
sudo apt install -y curl
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
```

Проверяем:

```bash
node --version
npm --version
```

---

📦 *Шаг 2. Установка OpenClaw*

Устанавливаем OpenClaw глобально через npm:

```bash
sudo npm install -g openclaw
```

Проверяем версию:

```bash
openclaw --version
```

---

📂 *Шаг 3. Первоначальная настройка (onboard)*

Вместо устаревшей команды `init` используйте `onboard`:

```bash
openclaw onboard
```

Ответьте на вопросы (выбирайте варианты для DeepSeek):

1. **Continue?** → `Yes`
2. **Setup mode** → `Manual`
3. **What to set up?** → `Local gateway`
4. **Workspace directory** → оставьте по умолчанию
5. **Model/auth provider** → `Custom Provider`
6. **API Base URL** → `https://api.deepseek.com`
7. **API key** → вставьте ваш ключ DeepSeek (получить на platform.deepseek.com)
8. **Endpoint compatibility** → `OpenAI‑compatible`
9. **Model ID** → `deepseek‑reasoner`
10. **Gateway port** → `18789` (по умолчанию)

После проверки ключа система создаст конфигурацию.

---

⚙️ *Шаг 4. Запуск Gateway*

Gateway — это фоновый сервис, который управляет агентами. Запускаем его:

```bash
openclaw gateway start
```

Проверяем статус:

```bash
openclaw gateway status
```

Если всё хорошо, увидите `Gateway is running`. По умолчанию Gateway использует порт **18789**.

---

🧪 *Шаг 5. Проверка работоспособности*

Проверим, что OpenClaw отвечает:

```bash
openclaw status
```

Запустим тестовый агент:

```bash
openclaw run "Напиши 'Hello, World!' на Python и выполни код"
```

Если агент выполнил запрос — система работает.

---

🔐 *Шаг 6. Настройка переменных окружения (опционально)*

Для хранения API‑ключей создадим файл `.env` в workspace:

```bash
cd ~/.openclaw/workspace
nano .env
```

Добавьте placeholder‑переменные:

```env
BINANCE_API_KEY=your_api_key_here
BINANCE_API_SECRET=your_api_secret_here
TELEGRAM_BOT_TOKEN=your_bot_token
```

---

🏗️ *Шаг 7. Знакомство с архитектурой OpenClaw*

Кратко о компонентах:

1. **Gateway** — центральный сервис.
2. **Agents** — экземпляры LLM с доступом к инструментам.
3. **Workspace** — директория проекта.
4. **Skills** — готовые модули для типовых задач.
5. **Tools** — инструменты (браузер, API, шелл).

🛠️ *Возможные проблемы и решения*

| Проблема | Решение |
|----------|---------|
| `openclaw: command not found` | Убедитесь, что пакет установлен глобально. Выполните `hash -r` или создайте симлинк вручную. |
| Gateway не запускается (`port already in use`) | Измените порт в `~/.openclaw/config.json` или освободите порт 18789 (по умолчанию). |
| Ошибка «Permission denied» | Проверьте права на `~/.openclaw`. |
| Агент не может выполнить код | Установите Python3. |
| Установился placeholder‑пакет `openclaw@0.0.1` | Обновите Node.js до ≥22.16.0, удалите и установите заново. |
| Команда `openclaw` не работает после установки | Выполните `hash -r` или создайте симлинк. |
