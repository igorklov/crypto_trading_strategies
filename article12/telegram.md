День 12: Фундаментальные факторы и их влияние при открытии ордеров

📅 20 апреля 2026

В предыдущих статьях мы построили торговую систему на технических индикаторах (RSI, SMA, Bollinger Bands, ADX). Но рынок движется не только графиками — новости, события, сентимент и макроэкономика часто вызывают резкие движения, которые индикаторы не ловят.

Сегодня добавим в систему фундаментальный анализ: научимся собирать данные из внешних источников, оценивать их влияние с помощью LLM и комбинировать с техническими сигналами для более умных решений.

➖➖➖

1/10 🤔 Зачем фундаментальные факторы в автоматической торговле?

Технический анализ отлично работает на истории, но «слеп» к внешним событиям:

• Внезапные новости: взлом биржи, регуляция, заявление инфлюенсера.
• Запланированные события: хардфорк, апгрейд, листинг.
• Макроэкономика: ставки ФРС, инфляция, геополитика.
• Социальный сентимент: волна обсуждений в Twitter, Reddit.
• On‑chain метрики: активные адреса, балансы на биржах.

Пример: 10 апреля 2026 SEC одобрила spot‑ETF на Ethereum → ETH +18% за 2 часа. Технические индикаторы не предсказали, но мониторинг новостей мог дать сигнал к покупке.

Наша цель — создать слой фундаментального анализа, который:
1. Собирает данные из множества источников.
2. Оценивает тональность и важность через LLM.
3. Генерирует дополнительные сигналы (buy/hold/sell).
4. Комбинирует с техническими сигналами в единой системе.

5/10 ⚖️ Интеграция с техническими сигналами

Теперь два независимых источника:
• Технический анализ (TA) → сигнал от индикаторов.
• Фундаментальный анализ (FA) → сигнал от новостей, сентимента.

Взвешенная модель:

| Фактор                 | Вес   |
|------------------------|-------|
| Технические индикаторы | 0.6   |
| Новости (срочные)      | 0.2   |
| Социальный сентимент   | 0.1   |
| On‑chain метрики       | 0.1   |

Алгоритм:
1. TA_score ∈ [-1, 1] (‑1 = strong sell, +1 = strong buy).
2. FA_score ∈ [-1, 1] на основе новостей и сентимента.
3. final_score = 0.6 * TA_score + 0.4 * FA_score.
4. Пороги: >0.3 → BUY, <‑0.3 → SELL, иначе HOLD.

➖➖➖

6/10 💻 Пример кода: гибридная система принятия решений

```python
class HybridDecisionSystem:
    def decide(self, symbol, ta_indicators, news_items):
        ta_score = self.get_ta_signal(ta_indicators)
        fa_score = self.get_fa_signal(news_items)
        final_score = 0.6 * ta_score + 0.4 * fa_score
        if final_score > 0.3:
            return "BUY", final_score
        elif final_score < -0.3:
            return "SELL", final_score
        else:
            return "HOLD", final_score
```

TA_signal вычисляется из RSI, ADX, Bollinger Bands, SMA.
FA_signal — средневзвешенная тональность новостей с учётом важности (high=1.0, medium=0.5, low=0.2).

8/10 ⚠️ Ограничения и риски

• Задержка данных — новости могут приходить с опозданием.
• Ложные новости и манипуляции — нужна проверка репутации источников.
• Переобучение на шум — рынок часто реагирует краткосрочно, затем откатывается.
• Стоимость API — качественные данные (Glassnode, Twitter API) часто платные.
• Сложность интерпретации — одна новость может толковаться по‑разному.

Рекомендации:
• Начинайте с бесплатных источников (Cryptopanic, CoinGecko).
• Тестируйте гибридную стратегию на демо‑счёте минимум месяц.
• Вводите «карантин» для новостей: игнорируйте сигналы первые 5 минут.
• Комбинируйте несколько источников для перекрёстной проверки.
